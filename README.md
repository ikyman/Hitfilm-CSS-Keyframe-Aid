# Basic Idea
My Video-Editing Program Is HitFilm Express. Video-Editing and Content Creation is a neat idea!

I am, however, slow as nails. This is a cradt! All the stops should be put out.

But do I really need to put out all the stops? Most of the time is spent adding the dozen or so keyframes to turn Idea into reality. Plus, if the AI Scuffs it up, fixing it shouldn't be too hard.

I have heard a lot of "App Ideas" that boils down to calling an AI API. Finally, though! One that **I** might use!

# XML to CSS, and vice-versa.

I start a new composite shot. I do not want to make this composite shot myself, so I will exit Hitfilm, and task the program with it.

This program shall Load in the XML. It shall identify the <CompositionAsset> Tags. This will provide a list of Composite shots to work on.

This program will also load the <AssetList> tag. This will provide a list of assets to use.

I write my Query. This gets translated into a "Help me with CSS Keyframes" Request.

This will return a Great big deal of CSS Keyframes, which can then be translated into the sort of XML Hitfilm uses.

(Oh, Yeah, Before I forget! This makes a save of the origional copy of my Mid-Editing file.)
